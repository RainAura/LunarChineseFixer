# 更新记录

## 1.0.12 - 2026-07-13

- 放弃旧版多层缓存修补，改为替换 Lunar `FontRenderer` 的核心绘制方法；
- 菜单、聊天、计分板统一使用 1.8.9 即时字体绘制，彻底绕过 Lunar 文字显示列表；
- 进入服务器或单人世界约 5 秒后自动执行一次资源刷新，修复重启后字体页再次错位；
- 内嵌 Java 17 字节码转换器、ASM 运行库和绘制模板，目标电脑无需额外安装 Java；
- 自动使用目标电脑的 Lunar Java，并保留原始缓存备份和写入后复检；
- 使用四份不同玩家缓存完成 ZIP、JAR、点号类名和标准类路径的修复及恢复测试。

## 1.0.11 - 2026-07-13

- 增加最终兜底注入，直接修改 Lunar `FontRenderer` 的字符串缓存入口；
- 每次文字绘制前销毁旧显示列表并强制缓存未命中，杜绝错误字形缓存跨绘制复用；
- 保留资源重载清理和 Unicode 纹理页强制绑定，形成三层兼容修复；
- 支持从 1.0.8、1.0.9、1.0.10 已修复缓存直接升级；
- 使用原始备份及三位不同玩家的 Lunar 缓存完成写入、复检和恢复测试。

## 1.0.10 - 2026-07-13

- 根据两份不同 Lunar 缓存及修复前原始备份，确认仅在资源重载时清缓存仍不足以阻止错字复发；
- 修补 Lunar 缓存文字渲染段，在每次绘制前强制同步 OpenGL 与 GlStateManager 的字体纹理状态；
- 解决中文字形 UV 坐标正确、但绑定到错误 Unicode 字体页造成的同位置错字；
- 保留原有资源重载清缓存补丁，形成缓存生命周期和实际纹理绑定两层修复；
- 支持从 1.0.9 已修复缓存直接升级，并对新增字节码逐项复检。

## 1.0.9 - 2026-07-13

- 扫描范围从固定 `bake.zip` 扩展到缓存目录内的 ZIP 和 JAR 归档；
- 同时兼容点号类名、带 `.class` 后缀及标准斜杠类路径；
- 同一缓存包存在多个 FontRenderer 条目时全部修复并逐项校验；
- 手动选择缓存子目录时自动向上定位 `.lunarclient` 根目录；
- 增加多种缓存布局的自动化修复、校验和恢复测试。

## 1.0.8 - 2026-07-13

- 修正问题根因：Lunar 1.8.9 的文字显示列表缓存没有在资源重载时清空；
- 自动扫描 `.lunarclient/offline/multiver/cache` 中可修复的 `bake.zip`；
- 为 `FontRenderer.onResourceManagerReload` 注入 `clearCaches()` 调用；
- 修改前自动创建独立原始备份，写入后再次解析字节码校验；
- 支持一键恢复 Lunar 原始缓存文件；
- Lunar 更新覆盖缓存后，可重新运行工具自动检测并再次修复；
- 不再通过关闭 OptiFine `Custom Fonts` 改变玩家字体设置。
